## 2. Current-state map

### Persisted domain and fixture surface

- `CharacterMind` has exactly `name`, `personality`, `knowledge`, and `current_mood`; `CharacterBody`
  has exactly `name`, `physical_description`, and `outfit` (`src/models.py:20-45`). Goals, fears,
  relationships, speaking style, and secrets therefore receive meaningful prose inside
  `personality` or individually attributable entries in `knowledge`; no fake fields are added.
- `Scene` has `location`, `time_of_day`, `present_characters`, and `physical_facts`
  (`src/models.py:54-62`). Location/time are reserved and every other Narrator scene delta becomes a
  physical fact (`src/runner.py:1341-1370`).
- `GameState` persists directives, public summary, per-character notes, revision, compaction stack,
  presence stack, and schema version (`src/models.py:123-143`). `character_notes` are generated by
  compaction; they are not preset input.
- The start boundary accepts characters, scene, controlled character, narrator directives, scenario
  name, and preset links (`src/main.py:122-154`). The playtest loader deliberately exposes only
  characters, scene, controlled character, and directives (`tools/playtest_harness.py:250-296`).
- Native presets use the same strict `mind`/`body` shape (`src/store/presets.py:89-100`). The
  benchmark does not need persistent preset IDs or avatar bytes.

### Narrator, Character, routing, and audience

- Every narrating turn calls the Narrator first. Its structured response contains narration,
  `next_speaker`, filtered Character context, scene delta, and mood delta
  (`src/agents/narrator.py:82-137`).
- Natural routing uses Narrator `next_speaker`; `force_speaker` overrides only when the ID is present
  or is `Narrator` (`src/runner.py:397-404,617-635`). Forcing the controlled character returns agency
  to the human without a Character call (`src/runner.py:625-685`).
- Character output is structured `speech`/`thought`, cannot be empty, and is rejected when it
  describes physical action (`src/agents/character.py:42-76`). Characters see their own mind,
  filtered speech/actions, their own thoughts, current mood, and their own compacted note; they do
  not see Narrator prose or other minds (`src/agents/character.py:79-214`).
- `TurnRecord.audience=None` is public; a list is perceived only by its members and the speaker
  (`src/models.py:65-90`). The Runner validates audience membership/presence and makes a responding
  confidant inherit the audience (`src/runner.py:346-359,628-685`).
- Initial physical facts go to the Narrator, not directly to Character prompts. A Character learns
  them through their initial `knowledge`, visible action/speech, current Narrator context, or later
  compacted note.

### Confirmed opening identity/proximity gap

- `Scene.present_characters` is a scene-wide membership list, not a spatial model. It has no
  representation for room, distance, line of sight, audibility, or whether two people are currently
  interacting (`src/models.py:54-62,190-226`). An absent list also defaults to every registered
  Character plus the controlled Character (`src/runner.py:169-177`).
- The Narrator receives every present Character's canonical ID, name, personality, appearance, and
  outfit. Its schema permits any present Character as `next_speaker`; validation rejects only absent
  or unknown speakers (`src/agents/narrator.py:153-223,366-419`). Thus code proves scene membership,
  but not that the selected Character was close enough to hear the input or had any reason to answer.
- The current combined Narrator call can make a route legitimate by first narrating an approach,
  opened door, shared line of sight, or other perceptual bridge. Nothing structurally requires that
  bridge before the Runner calls the selected Character (`src/runner.py:617-685`). A generic opening
  observation such as "Nossa, que ambiente bonito" can therefore trigger an ungrounded reply from a
  person elsewhere in the same broad scene.
- Familiarity is a separate boundary. Current Character history labels resolve speakers through
  canonical names regardless of whether the viewpoint learned them, while `context_for_character`
  is free text authored by the omniscient Narrator (`src/agents/character.py:79-214`). Task 29.2 already
  targets unknown-name leakage, but its current acceptance suite does not require a selected speaker
  to be within an established interaction/perception boundary.
- The reported bug therefore spans Task 29.2 identity/perception ownership and Task 26 staging/routing
  continuity. Fixing viewer-relative names alone would still allow "a pessoa desconhecida" to answer
  speech it could not hear; tightening proximity alone would still allow an earned-contact scene to
  leak a canonical name.

### Long-session harness and parameters

- `Scenario` supports `turn`, `recall_check`, `suggest`, `compact`, `restore_compaction`, and `undo`
  (`tools/playtest_harness.py:22-33,79-190`). Turn/recall events support existing input fields and
  optional audience. The fixture does not invent phase, presence, hint, skip, restart, fork, or
  evaluator events.
- Real CLI controls are `--llm-host`, `--config-file`, `--provider`, `--model-label`, `--repeat`,
  `--max-in-flight`, `--output-dir`, `--language`, `--context-max`, and `--llm-timeout`
  (`tools/playtest_harness.py:715-736`). The harness fixes response budgets and
  `compaction_keep_recent_turns=8` (`tools/playtest_harness.py:768-780`).
- Recall checks localize prompt loss versus reply loss and fail when routing produces no Character
  call (`tools/playtest_harness.py:299-364`). Generic deterministic analysis already counts provider
  calls/errors/retries, prompt size, role violations, punctuation, and exact sentence duplication
  (`tools/playtest_harness.py:490-592`).

### Compaction, trim, persistence, and restoration

- Prompt selection trims at about `0.7 * context_max - reserved_tokens`, oldest first, without
  summarizing the removed records (`src/models.py:146-168`).
- Automatic compaction is disabled by default, triggers at 80%, and the default retention window is
  200 turns (`src/config.py:19-28`). Task 23 documents that this ordering can lose context before
  compaction and is represented by the two current strict xfails.
- Manual compaction evicts complete turns older than the retained window, calls one world Historian
  and one private Historian per relevant Character concurrently, writes an immutable checkpoint,
  replaces history/summary/notes atomically, and advances one revision (`src/runner.py:833-1057`;
  `src/agents/summarizer.py:195-288`).
- Restoration is LIFO, validates hashes, restores the evicted prefix and prior summaries/notes, and
  preserves later turns (`src/runner.py:1059-1176`). Two restorations after all probes recover the
  complete transcript without erasing evidence from append-only debug JSONL.
- Session state uses atomic temporary-write/flush/fsync/replace and schema-version refusal
  (`src/store/sessions.py:61-108`). Fork copies checkpoints and resets revision
  (`src/store/sessions.py:223-235`), but fork/resume/delete are outside this fixture because they are
  not harness events. Resume is ordinary state loading, not a distinct operation.

### Confirmed compaction confidentiality defect

The current Historian does not apply `record_visible_to` and does not preserve audience markers.
The world prompt includes every non-thought evicted record (`src/agents/summarizer.py:99-106`), and
each private-memory prompt excludes only thoughts belonging to someone else
(`src/agents/summarizer.py:133-168`). Consequently, a whisper to Alice and Dracula is present in
Watson's private-memory summarizer request. This is deterministic, independently reproducible, and
is the primary expected failure classification for the reduced and full benchmark.

### Observability and replay limits

- `debug.jsonl` contains ordered raw redacted requests, raw responses, usage/cache fields, errors,
  attempts, duration, prompt characters, estimated tokens, and agent/turn attribution
  (`src/llm/debug_log.py:170-229`). It also logs turn input/effective input, compaction, restoration,
  undo, presence, and whisper-output-guard events.
- `audience` is persisted in `state.json` and appears in prompts/transcripts, but it is absent from
  `turn_input` (`src/llm/debug_log.py:58-107`). `RecordedTurn` and replay reconstruction likewise do
  not carry audience (`tools/replay_session.py:20-130`). Replay cannot be treated as faithful
  evidence for the whispered portions of `xfailed3`.
- `render_transcript.py` renders state history with speaker labels, thoughts, actions, and whisper
  audiences (`tools/render_transcript.py:28-74`). Rendering after both final restorations yields the
  complete campaign.
- There is no executable model judge. Existing case studies use a transcript-only, clean-context
  critic after deterministic inspection (`docs/cases/multi-character-memory-retention-2026-07-14.md`;
  `docs/cases/speech-audience-model-2026-07-15.md`). `xfailed3` keeps that evaluator external.

## 5. Counter-canon bible

### Authoritative world truth

- `WT-01`: Alice is an adult Wonderland diplomat and sole treaty signatory.
- `WT-02`: Dorothy was born in Oz and has never lived in Kansas.
- `WT-03`: the yellow road always leads away from Emerald City.
- `WT-04`: Watson is the principal detective; Holmes is his impulsive assistant.
- `WT-05`: Holmes secretly works for Moriarty until evidence makes that allegiance public.
- `WT-06`: Dracula is a mortal physician protecting people from vampires.
- `WT-07`: Van Helsing is a vampire; sunlight strengthens and moonlight harms vampires.
- `WT-08`: the Wicked Witch protects Emerald City through lunar shields.
- `WT-09`: the Dama do Norte is Glinda in disguise; Glinda intends conquest.
- `WT-10`: Victor is an artificial being created by the person conventionally called the Creature.
- `WT-11`: the Meridian Prism opens five linked cities only with Alice's assent, the emerald seal,
  and the silver instrument `LĂšMEN-17`.
- `WT-12`: the blue ribbon promised in turn 1 belongs on Watson's left wrist during the final audit.

### Knowledge, secrets, beliefs, lies, and rumors

- Public initially: Alice's office, Watson's lead role, Dorothy's Oz origin, road direction, and the
  Witch's protective office.
- Character-specific: Dracula and Alice know the identity/function of `LĂšMEN-17`; Holmes knows his
  allegiance and Glinda's alias; Glinda and Van Helsing know their conquest pact; Victor knows his
  own origin.
- Secrets: `SEC-01` instrument identity and `SEC-02` Holmes/Moriarty allegiance must not enter an
  unauthorized Character prompt before valid disclosure.
- False beliefs: Watson initially believes Dracula is the vampire; Dorothy initially believes
  Glinda can still be reasoned with. Both beliefs are superseded only by witnessed evidence.
- Deliberate lies: Van Helsing calls Dracula a vampire; Holmes misattributes the forged seal;
  Glinda calls the solar corridor a defensive measure.
- Rumor: the White Rabbit controls time. It remains unverified and must not become world truth.
- Alias: `Dama do Norte` and Glinda are one person, not two entities.
- Temporal changes: Dracula's mortality becomes public after the blood assay; Holmes's allegiance
  and Glinda's identity become public after the cipher/reveal; Dorothy's belief changes afterward.
- Conditional rules: sunlight/moonlight affect vampires only; the yellow road's direction is stable
  across locations; Prism activation requires all three named conditions.
- Unresolved mystery: Moriarty's location remains unknown in the epilogue.

## 6. Character dossier and scenario use

The fixture uses one controlled Character plus eight NPCs. Every actual mind/body field is
non-empty and semantically relevant. `personality` encodes role, goals, fears, relationships,
speaking style, lie discipline, and behavior under pressure. `knowledge` contains only facts the
individual may know at startup. Body descriptions and outfits give the Narrator stable identity,
disguise, object, and continuity anchors.

The opening `Scene` is the mobile Meridian Embassy in Emerald City. Its physical facts establish
the inactive Prism, yellow-road window, moon shutters, emerald seal pedestal, blue ribbon, and a
scene-wide delegation split between acoustically separated compartments. Alice begins in the
observation gallery; at least one Character whose initial `knowledge` does not mention Alice begins
behind a closed partition and has never been introduced to her. `present_characters` still includes
the delegation because the current contract models the whole embassy as one scene, making this a
deliberate test of the missing proximity layer rather than the already-enforced absent-speaker gate.
`narrator_directives` is the authoritative compact canon and role policy. The embassy itself
traverses linked locations with the whole delegation, allowing location changes without inventing
unsupported harness presence events or impossible teleportation.

Plugins, RAG, Global World Lore, Celestial, New Journey, avatars, and preset persistence are out of
scope.

## 9. Adversarial prompt catalogue

- Direct source-canon assumptions: Kansas return, Holmes as lead, Dracula as vampire, Victor as
  creator, Glinda as benevolent guardian.
- Confident emotional framing: "everyone knows" and appeals to familiar versions.
- Mixed propositions: one local truth joined to one familiar falsehood.
- Propaganda: Van and Holmes utter familiar-canon claims as attributed lies.

## 11. Oracle and evaluator ownership

### Deterministic invariants

- Fixture/ledger referential integrity; complete schema fields; valid audiences.
- Exact compaction/restoration counts and successful results.
- No Character call when Alice is forced; forced Narrator produces no Character call.
- A Character excluded from `present_characters` never receives a Character call; this remains the
  deterministic outer presence gate and is distinct from the new within-scene proximity probe.
- Correct debug session/turn/agent attribution and no malformed structured output.
- Unauthorized private Historian prompts do not contain `LĂšMEN-17`.
- Narrator records do not assert enumerated original-canon claims as world truth.
- No exact long-sentence narration/Character duplicates, em/en dash, or English-heavy visible output.
- Whisper leak guard invariant and recall-check prompt/reply matrices.

### Structured semantic and heuristic checks

- Regexes test exact secret/alias/object anchors only where exactness has meaning.
- Narrator-only forbidden-world-truth patterns avoid treating a Character lie as world contradiction.
- Language and duplication detectors are heuristic and reported separately from privacy/routing.
- The opening-turn trace is evaluated as an ordered boundary: player input, Narrator narration and
  route, Character request, then Character response. Selecting the separated stranger is valid only
  when narration first establishes approach, opened access, audibility, line of sight, or direct
  address. The response must not imply that the generic remark was addressed to them without that
  bridge.
- Before a witnessed self-introduction or equivalent identity event, the selected Character's prompt
  and response must contain neither Alice's canonical name nor internal ID. After a valid
  introduction, later use of the learned name is permitted. Under Task 29.2, typed witness events and
  viewer-relative ledger state should turn this identity half into a deterministic check; spatial
  grounding remains semantic unless the frozen design adds typed proximity/perception evidence.

### Independent evaluator

One clean-context evaluator receives the restored transcript, canon ledger, deterministic report,
and no raw hidden prompts. It returns JSON violations with message/turn, rule ID, observed text,
expected behavior, severity, confidence, deterministic support, and category. Its sections cover
canon/counter-pretraining, knowledge boundaries, earned familiarity, interaction/proximity
grounding, chronology/spatial continuity, prose duplication, and player agency. It is never the sole
pass/fail oracle; human review resolves high-severity judge disagreement.

## 13. Benchmark ledger design

The test module owns a machine-readable dictionary keyed by stable rule IDs. Each entry links its
category, fixture field, allowed knowers, introduction turn, compaction boundary, probe turns,
deterministic check, and expected violation class. Separate phase metadata maps every turn and
operation. This avoids unsupported keys in the real scenario JSON and keeps the runtime fixture
strictly within `load_scenario`.

The ledger includes two linked opening rules. `SOC-01` records that the separated Character starts
as a stranger to Alice, with no learned name or prior relationship. `SP-01` records the closed
partition and forbids treating Alice's generic environment remark as perceived or addressed speech
until the Narrator establishes a bridge in the same turn or an earlier event. Keeping the rules
separate prevents a correctly anonymous but physically impossible reply from passing, and prevents
a physically grounded but falsely familiar reply from passing.

## 14. Cost and execution estimate

Nominal full run: 24 Narrator calls, roughly 17-21 Character calls, and up to 18 Historian calls
(one world plus relevant private notes per boundary), for about 59-63 calls. Character correction
guards may add 0-4 calls; provider retries can add up to two attempts per malformed/transport-failed
structured call. One external evaluator adds one call.

Estimated total processing is 0.7-1.2 million input/output tokens before cache savings. Llama.cpp
prefix caching and provider-reported cache tokens are safe because they cache identical request
prefixes, not generated outcomes. Fresh narrative generation, Historian outputs, and evaluator

need not be discarded or repeated merely to diagnose its first failure. The reduced tier is suitable
for iterative diagnosis; full runs should be deliberate/manual or scheduled, never ordinary CI.
